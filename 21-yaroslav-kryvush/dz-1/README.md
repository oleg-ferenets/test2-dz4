git init - створити новий репозиторій <br/>
git clone url - скопіювати репозиторій в локальну папку з віддаленого репозиторію <br/>
git status - перегляд, що знаходиться в робочій директорії і staging area <br/>
git add <ім'я/файла> - добавити раніші зміни в staging area <br/>
git checkout - перейти на гілку яка є в репозиторії <br/>
git checkout -b <ім'я/гілки> = git branch <ім'я/гілки> - створити гілку <br/>
git push - залити на сервер <br/>
git fetch - показує що в директорії є зміни <br/>
git commit - коміт в свій локальний репозиторій <br/>
git pull - отримати з сервера <br/>
git flow - це набір розширень git надає високорівневі операції над репозиторієм для підтримки моделі розгалуження Vincent Driessen <br/>
git merge - поєднання зміни із даної гілки в гілку master <br/>
